Modelos de Cloud (Computación en la Nube)

A continuación, se definen los modelos de cloud, que se refieren a los tipos de servicios de computación en la nube.

SaaS (Software as a Service)

Servicio de software proporcionado a través de internet, donde el usuario accede a la aplicación sin gestionar infraestructura o software subyacente.

PaaS (Platform as a Service)

Plataforma proporcionada por el proveedor para desarrollar, probar y desplegar aplicaciones sin preocuparse por la infraestructura subyacente.

IaaS (Infrastructure as a Service)

Servicio que ofrece recursos de computación virtualizados, como servidores, almacenamiento y redes, para que el usuario gestione el software y la plataforma.

Modelos de Entrega (Delivery Models)

Los modelos de entrega se refieren a estrategias para liberar software de manera eficiente. A continuación, ejemplos de cada uno.

Blue-Green Deployment

En este modelo, se mantienen dos versiones del entorno: "azul" para la producción actual y "verde" para la nueva. Una vez probado, se cambia el tráfico al nuevo entorno. Ejemplo: Una aplicación web con cambios en la interfaz; se despliega en verde, prueba, y luego se conmuta.

Canary Deployment

Libera cambios a un pequeño porcentaje de usuarios para probar antes de un despliegue completo. Ejemplo: Una actualización de una app móvil que se lanza al 10% de los usuarios, se monitorea, y si es exitoso, se expande.

Rolling Deployment

Actualiza instancias una por una o en grupos pequeños para minimizar downtime. Ejemplo: Un servidor web con múltiples instancias; se actualiza una instancia a la vez mientras las otras manejan el tráfico.

Modelos de Despliegue (Deployment Models)

Los modelos de despliegue describen cómo se alojan y administran los servicios en la nube.

On-Premise

El hardware y software se alojan en las instalaciones físicas de la empresa, ofreciendo máximo control y seguridad.

Cloud

Los recursos se hospedan en centros de datos remotos administrados por proveedores de nube.

Híbrido

Combina on-premise y cloud, permitiendo datos sensibles localmente y escalabilidad en la nube.

Proveedores de cloud que conozco: Amazon Web Services (AWS), Microsoft Azure, Google Cloud Platform (GCP).

Explicación de Comandos de Git Utilizados

En este proyecto, se utilizaron los siguientes comandos de Git para crear el repositorio, rama, subir cambios y añadir miembros.

  • git init: Inicializa un nuevo repositorio de Git en el directorio local.
  • git checkout -b tallerEvaluativo1: Crea y cambia a una nueva rama llamada "tallerEvaluativo1".
  • git add .: Añade todos los archivos modificados al staging area para el próximo commit.
  • git commit -m "Mensaje": Confirma los cambios con un mensaje descriptivo, creando un snapshot del estado actual.
  • git remote add origin [URL]: Añade el repositorio remoto como origen.
  • git push -u origin tallerEvaluativo1: Sube la rama y cambios al repositorio remoto, estableciendo el upstream.

Para añadir miembros en GitHub, se debe ir a Configuración > Miembros > Invitar a colaboradores, e ingresar el email (cespedeslina10@gmail.com).
