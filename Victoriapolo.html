Definición de Cloud Computing

Cloud Computing, o Computación en la Nube, se refiere a la entrega de servicios de computación a través de Internet, incluyendo servidores, almacenamiento, bases de datos, redes, software y mucho más. Permite el acceso a recursos tecnológicos sin necesidad de infraestructura propia, pagando solo por lo que se usa. Esto aporta flexibilidad, escalabilidad y reducción de costos.

Modelos de Entrega en Cloud

Los modelos de entrega definen cómo se proporcionan los servicios en la nube. A continuación, se explican los principales: Software as a Service (SaaS), Platform as a Service (PaaS) e Infrastructure as a Service (IaaS), con ejemplos.

1. Software as a Service (SaaS)

El proveedor ofrece el software completo a través de la nube, y los usuarios lo usan sin preocuparse por la infraestructura subyacente. El proveedor maneja todo, desde el desarrollo hasta el mantenimiento.

  • Ejemplo: Servicios como Gmail (Google), SalesForce, o Microsoft Office 365. Los usuarios acceden al correo electrónico o las herramientas de productividad directamente desde el navegador, sin instalar nada localmente.

2. Platform as a Service (PaaS)

Providencia una plataforma completa para desarrollar, probar e implementar aplicaciones, incluyendo herramientas de desarrollo y lenguajes. Los desarrolladores no gestionan la infraestructura física, pero controlan sobre el código y aplicaciones.

  • Ejemplo: Heroku, Google App Engine o Azure App Service. Un desarrollador puede subir su aplicación Node.js y el PaaS se encarga de escalarla automáticamente sin preocuparse por servidores.

3. Infrastructure as a Service (IaaS)

Ofrece recursos básicos de infraestructura computacional a través de la nube, como máquinas virtuales, almacenamiento y redes. El usuario tiene control total sobre la configuración del software y el sistema operativo.

  • Ejemplo: Amazon Web Services (AWS EC2), Microsoft Azure Virtual Machines o Google Compute Engine. Un administrador puede lanzar una instancia de servidor virtual para ejecutar una base de datos, configurándola según necesidades específicas.

Modelos de Despliegue en Cloud

Los modelos de despliegue determinan cómo se distribuyen los recursos y quién puede acceder a ellos. Los principales son: Nube Pública, Nube Privada y Nube Híbrida.

1. Nube Pública

Los recursos son proporcionados por un proveedor tercero y están disponibles para cualquiera en Internet. Es rentable y escalable, pero con menos control sobre seguridad y regulación.

  • Ejemplo: AWS, Microsoft Azure o Google Cloud Platform, donde empresas como Netflix despliegan aplicaciones para acceso global.

2. Nube Privada

La infraestructura está dedicada a una sola organización, operada por la empresa o un proveedor externo. Ofrece mayor seguridad y control, pero requiere inversión inicial.

  • Ejemplo: VMware vSphere para bancos, donde se aloja datos sensibles de clientes sin compartir infraestructura con otros.

3. Nube Híbrida

Combina nubes públicas y privadas, permitiendo mover datos y aplicaciones entre ambas según necesidades. Ideal para balancear costos, seguridad y escalabilidad.

  • Ejemplo: Una empresa como Amazon podría usar nube privada para datos críticos y pública para picos de demanda, integrándose con AWS Hybrid Cloud.

Proveedores de Cloud Conocidos

Algunos proveedores de servicios en la nube que conozco incluyen: Amazon Web Services (AWS), conocido por su amplia gama de servicios y escalabilidad; Microsoft Azure, integrado con herramientas de Microsoft; Google Cloud Platform (GCP), fuerte en análisis de datos y machine learning; y otros como IBM Cloud y Oracle Cloud, que ofrecen soluciones especializadas.

Explicación de Comandos Git Utilizados

Durante el proceso, se utilizaron los siguientes comandos de Git para gestionar el repositorio y subir los cambios. A continuación, una explicación de cada uno:

  • git init: Inicializa un nuevo repositorio Git en el directorio local, creando la estructura para versionado de archivos.
  • git add . o git add VictoriaPolo.html: Agrega el archivo al área de staging, preparándolo para el commit.
  • git commit -m "mensaje": Crea un commit con los cambios modificados, guardando una instantánea del proyecto con un mensaje descriptivo, por ejemplo "Añadiendo página inicial".
  • git branch tallerEvaluativo1: Crea una nueva rama llamada "tallerEvaluativo1" para trabajar en paralelo.
  • git checkout tallerEvaluativo1: Cambia a la rama especificada para trabajar en ella.
  • git remote add origin [URL del repositorio]: Agrega el repositorio remoto con una URL específica como origen para sincronizar cambios.
  • git push -u origin tallerEvaluativo1: Sube los commits de la rama local al repositorio remoto, configurando la rama upstream para futuros pushes.
  • git status: Verifica el estado del repositorio, mostrando archivos modificados, staged o committed.
  • git log: Muestra el historial de commits para revisar cambios realizados.
